깃 설치
터미널에서 git config --global user.name "깃 허브 이름이랑 똑같이"
	git config --global user.email "깃 허브 이메일"
1. github 가입하고 repository (저장소) 하나 만들기
2. 작업폴더를 하나 만든 다음 터미널에서 열어서 git init 하기 (로컬 리포지토리 생성 법임)
3. github.com은 이제 기본 브랜치 이름을 master이 아니라 main으로 사용하라고 강요함, 그래서 로컬 작업 폴더에 있는 기본 브랜치 이름도 main으로 변경해주는 게 나을듯 터미널 열어서 git branch -M main을 입력하면 기본 브랜치 이름이 main으로 변경됨

로컬 -> 원격 저장소 업로드 하고 싶으면 작업 폴더에서 터미널 켜서 git push -u 원격저장소주소 main 입력

-u 옵션은 방금 입력한 주소 기억해두라는 뜻.

git push -u origin main 으로 사용가능

origin은 변수 같은 느낌 변수 목록 살펴보고 싶으면 git remote -v 입력

저장소에 올리지 않는 파일들은 .gitignore 파일들을 명시.

협업 git clone 원격저장소주소 : 원격 저장소에 있던 내용(파일)을 그대로 복제

복제한 뒤에 팀원도 폴더 열어서 코드짜고 commit 하고 git push 하면 되는데, 그 팀원도 git hub아이디가 있어야 하고, 그 팀원의 아이디를 Collaboratiors 메뉴에 등록해놔야 협업 가능

git push는 맘대로 할 수 있는 게 아님, 갑자기 다른 팀원이 만든 파일이 원격 저장소에 생기면 git push 못함. 

원격 vs 로컬 내용이 다르다면 로컬 저장소에서 git push가 안됨

git pull 원격저장소주소 -> 원격 저장소에 있던 모든 내용 가져와서 로컬 저장소에 합치기 하고 나서 하면 됨
